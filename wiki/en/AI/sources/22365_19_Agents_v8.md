## Summary
This whitepaper defines [[en/AI/Engineering/Agent_Engineering/Agent_Engineering|Agent Engineering]] as "a GenAI model-based system that combines access to external information with reasoning and logic." The core message is that while a model alone is confined to static knowledge and single-turn reasoning, an Agent interacts with the external world through tools, manages session history, and performs multi-step tasks using cognitive architectures (Cognitive Architecture) such as ReAct and CoT.

## Core Definitions
- **Agent**: An application that autonomously achieves goals by observing the environment and acting with tools.
- **Cognitive Architecture**: A reasoning pattern combining Model + Tools + Orchestration Layer.

## Agent vs Model Comparison
| Aspect | Models | Agents |
|---|---|---|
| Knowledge | Fixed at training time | Expanded through Tools |
| Reasoning | Single-turn inference | Session history + multi-step loop |
| Tool Use | None | Native |
| Cognitive Architecture | None | CoT, ReAct, LangChain, etc. |

## Cognitive Architecture (Reasoning Framework)
- **ReAct** ([[en/AI/Engineering/Agent_Engineering/Planning_and_Reflection|Planning & Reflection]]): Thought → Action → Action Input → Observation → Final Answer loop. Many cases of surpassing SOTA.
- **Chain-of-Thought (CoT)**: Sub-techniques — self-consistency, active-prompt, multimodal CoT.
- **Tree-of-Thoughts (ToT)**: Lookahead and multiple reasoning path exploration.
- ReAct walkthrough example: User Query → Question → Thought → Action (Flights/Search/Code/None) → Action Input → Observation → Repeat → Final Answer.

## Tools — 3 Types
### Extensions
- A bridge between Agent and API. **Agent-side execution**.
- Teaches the Agent how to use them with examples and parameter specifications.
- 1-to-many: A single Agent can handle multiple Extensions/APIs.
- Example: `Extension.from_hub("code_interpreter")` → calls `generate_and_execute` operation.

### Functions (Function Calling)
- Self-contained code modules. **Client-side execution**.
- The model outputs only the function name and arguments, while the actual call is executed by the client ("API stubbing").
- When to use: API middleware required, security constraints, batch/HITL timing, post-processing required.
- Code example: `gemini-2.0-flash-001`, `automatic_function_calling=disabled`, `tool_config=ToolConfig(function_calling_config=FunctionCallingConfig(mode='ANY'))`.
- `display_cities(cities, preferences)` function returns `{cities: ["Aspen", "Park City", "Whistler"], preferences: "skiing"}` JSON.

### Data Stores
- Real-time data based on vector databases. Uses PDFs, spreadsheets, websites, HTML, and structured/unstructured formats in their original forms.
- Connected via [[en/AI/sources/whitepaper_emebddings_vectorstores_v2|Embeddings & Vector Stores]].

### RAG Lifecycle
User Query → Embedding model → Matching in vector DB using **ScaNN** algorithm → Text retrieval → Agent forms a response ([[en/AI/Engineering/Context_Engineering/Retrieval_Strategies/RAG/RAG|RAG]]).

### Tools Comparison Table
| Tool | Execution Location | Typical Use Case |
|---|---|---|
| Extensions | Agent-side | Multi-hop API planning |
| Function Calling | Client-side | Security/batch/offline API |
| Data Stores | Agent-side | RAG (web/structured/unstructured) |

## Targeted Learning
- **In-context Learning**: Prompt + tools + few-shot at inference (ReAct example).
- **Retrieval-based In-context Learning**: Vertex AI Example Store, RAG-based dynamic examples.
- **Fine-tuning**: Pre-training for a specific domain ([[en/AI/Engineering/Model_Engineering/Full_Fine-Tuning|Full Fine-Tuning]]).
- Chef analogy: Given recipe / well-stocked pantry / culinary school.

## LangChain Quickstart
- `langgraph.prebuilt.create_react_agent`, `SerpAPIWrapper`, `GooglePlacesTool`, `ChatVertexAI(model="gemini-2.0-flash-001")`.
- Multi-step query: "Who did the Texas Longhorns play last week? What is the address of the opponent's stadium?" → Search → Places → "Sanford Stadium, 100 Sanford Dr, Athens, GA".

## Production (Vertex AI)
- Vertex AI Agent Builder, Vertex Extensions, Vertex Function Calling, Vertex AI Example Store.
- Mentions "agent chaining" and "mixture of agent experts" patterns in the conclusion.

## Key Takeaways
- The quality of an Agent is proportional to "the model's reasoning/acting capabilities" and "how well the tools are defined."
- Tools are categorized into three major classifications: Extensions, Functions, and Data Stores.
- Learning strategies increase in cost/depth in the order of in-context → retrieval-based → fine-tuning.
